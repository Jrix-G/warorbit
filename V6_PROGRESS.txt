ÉTAPE 1 : search.py
─────────────────────────────────────────────────────
Créer search.py avec :

  def generate_candidates(state, me, n_candidates) -> list of action_lists
    5 stratégies cycliques (i % 5) :
      0 = Agressif : ratio 0.65-0.80, cible ennemis les plus proches
      1 = Défensif : ratio 0.30-0.50, garde les planètes menacées
      2 = Production : ratio 0.50-0.65, cible neutres haute production
      3 = Comète : rush comète active si fenêtre (abs(turn - comet_turn) < 15)
      4 = Perturbation : varie légèrement le meilleur plan précédent (±20%)

  def _generate_action_set(state, me, strategy, best_actions=None) -> list
    Pour chaque planète du joueur :
      - Filtrer planètes avec ships > 5
      - Calculer angle via math.atan2 vers cible choisie
      - Retourner [[from_id, angle, n_ships], ...]

  def _heuristic_actions(state, player) -> list
    Version simplifiée V5 pour simuler adversaires :
      - Pour chaque planète du joueur avec ships > 5
      - Cible = max(production * 30 - distance * 0.5) parmi non-alliés
      - Envoie 60% des ships
      - Très rapide (appelé des centaines de fois dans beam search)

  def beam_search(state, time_budget=0.85, evaluator=None) -> list
    - t_start = time.time()
    - me = state.player
    - n_candidates adaptatif (50 si budget normal, moins si budget réduit)
    - horizon adaptatif (20 tours normalement)
    - Pour chaque candidat :
        * Simuler `horizon` tours avec step_inplace()
        * Adversaires simulés avec _heuristic_actions
        * Score = Σ evaluate_state * GAMMA^t (GAMMA=0.97) + 5x score final
        * Garder meilleur
    - Check temps à chaque candidat (break si t > time_budget * 0.9)
    - Retourner best_actions (jamais None, fallback [] si rien)

  IMPORTANT pour performance :
    - Utiliser step_inplace() pas step() pour éviter la copie
    - Copier state.copy() UNE FOIS par candidat, puis step_inplace en boucle
