CE QUE TU DOIS PRODUIRE
================================================================================

Tu dois créer les fichiers suivants, DANS CET ORDRE :

1. sim.py
   Simulateur numpy du jeu Orbit Wars.
   Classe GameState, fonctions step(), state_from_obs(), fleet_speeds(),
   update_planet_positions(), _resolve_arrivals().
   CRITIQUE : doit être fidèle aux règles exactes du moteur Kaggle.
   Cible performance : >1000 steps/seconde.

2. eval.py
   NumpyEvaluator (réseau 3 couches, ~900 paramètres).
   Fonctions extract_features() (24 dims), evaluate_state(),
   encode_evaluator(), decode_evaluator().
   Doit fonctionner avec numpy uniquement, zéro dépendance ML externe.

3. search.py
   Beam search avec budget temps strict.
   Fonctions beam_search(), generate_candidates(), _heuristic_actions().
   Budget : 0.85 secondes max par tour.
   Génère 5 types de stratégies (agressif, défensif, production, comète, perturbation).

4. train_v6.py
   Boucle self-play AlphaGo-style.
   Fonctions generate_self_play_games(), train_evaluator_backprop(),
   train_evaluator_cma(), classe League, fonction principale train_v6().
   Doit pouvoir tourner sur Kaggle Notebooks (30h/semaine gratuit).

5. test_sim.py
   Tests unitaires du simulateur.
   Valide que sim.py produit les mêmes résultats que kaggle_environments
   sur 10 parties complètes. Si un seul test échoue, corriger sim.py.

6. bot_v6.py
   Agent final tout-en-un pour soumission Kaggle.
   Contient tout le code nécessaire inline (sim + eval + search).
   Poids de l'évaluateur encodés en base64 directement dans le fichier.
   Fallback sur logique V5 si budget temps dépassé.
   Fichier unique, aucune dépendance externe sauf numpy.

================================================================================
CONTRAINTES ABSOLUES
================================================================================

1. NUMPY UNIQUEMENT dans bot_v6.py
   PyTorch, TensorFlow, scikit-learn = INTERDITS dans le fichier de soumission.
   Raison : non disponibles dans l'environnement d'évaluation Kaggle.
   numpy, math, time, collections, base64, io = OK.

2. BUDGET TEMPS : 1 seconde par tour
   Le beam search doit finir en <0.85s.
   Si remainingOverageTime < 5s → fallback V5 immédiat.
   Toujours retourner quelque chose, jamais de timeout.

3. FIDÉLITÉ DU SIMULATEUR
   La physique doit être exacte :
   - Vitesse flottes : speed(n) = 1 + 5 * (log(n)/log(1000))^1.5
   - Orbites : angle = arctan2(dy,dx) + angular_velocity * step
   - Combat : plus grand gagne, différence survit, égalité → neutre 0 ships
   - Production : uniquement sur planètes avec owner >= 0

4. FORMAT DE SORTIE
   return [[from_planet_id, angle_radians, n_ships], ...]
   Identique à V5. Jamais de None, jamais de crash.

5. ORDRE D'IMPLÉMENTATION OBLIGATOIRE
   sim.py → eval.py → search.py → test_sim.py → train_v6.py → bot_v6.py
   Ne pas sauter d'étape. Valider chaque module avant de passer au suivant.

================================================================================
CONTEXTE TECHNIQUE SUPPLÉMENTAIRE
================================================================================

Observation Kaggle (format entrée de agent(obs, config)) :
  obs.planets   → [[id, owner, x, y, radius, ships, production], ...]
  obs.fleets    → [[id, owner, x, y, angle, from_planet_id, ships], ...]
  obs.player    → int (0-3)
  obs.angular_velocity → float (rad/tour)
  obs.initial_planets  → positions initiales planètes
  obs.comets    → [{planet_ids, paths, path_index}, ...]
  obs.comet_planet_ids → list[int]
  obs.remainingOverageTime → float (secondes)
  obs.step      → int (tour actuel 0-499)

Constantes physiques :
  BOARD_SIZE = 100.0
  SUN_X, SUN_Y = 50.0, 50.0
  SUN_RADIUS = 10.0
  MAX_SPEED = 6.0
  ROT_LIMIT = 50.0  (orbite si dist_centre + rayon < 50)
  TOTAL_TURNS = 500
  COMET_TURNS = [50, 150, 250, 350, 450]
